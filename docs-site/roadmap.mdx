We build in the open. Priorities below are ordered by impact on SimAPI's role as
the validation layer for engineering simulation.

## In progress

<CardGroup cols={2}>
  <Card title="Durable job queue" icon="database">
    Redis-backed queue and persistent storage so results survive restarts and
    scale across replicas.
  </Card>
  <Card title="Baseline & regression detection" icon="chart-line">
    First-class baselines and automatic flagging when a run drifts.
  </Card>
</CardGroup>

## Next

<CardGroup cols={2}>
  <Card title="Organizations & RBAC" icon="users">

  </Card>
  <Card title="Usage dashboard & billing" icon="credit-card">
    Real-time usage, quotas, and self-serve billing.
  </Card>
  <Card title="Webhooks" icon="webhook">
    Push validation events to your systems.
  </Card>
  <Card title="JS / TS SDKs" icon="js">
